# Getting Started

Manti UI ships as a small set of packages, but for a React app you install just
**one** — the renderer. Everything else comes along automatically.

## Install

```bash
npm install @manti-ui/react
```

That is the only package you install. `@manti-ui/styles`, `@manti-ui/tokens`, and
the behavior machines (`@manti-ui/folds`) are declared as dependencies of
`@manti-ui/react`, so your package manager pulls them in automatically — you never
run a separate `install` for `@manti-ui/styles`.

> Using pnpm or Yarn? `pnpm add @manti-ui/react` or `yarn add @manti-ui/react`.

## Import the styles once

`@manti-ui/styles` is already in your `node_modules` (it arrived with
`@manti-ui/react`), but the React package ships **no** CSS side effects — so you
import the stylesheet yourself, once, at your app entry. Keeping the import
explicit leaves you in control of order (which matters when you also use
Tailwind), and this single line sets up the cascade layers, design tokens,
component styles, and motion tiers.

```ts
// main.tsx / app entry
import '@manti-ui/styles/index.css';
```

## Set the theme

Theming is driven by a `data-theme` attribute on `<html>` (or any ancestor), with
native `light-dark()` resolving the right values. Set it to `light` or `dark`:

```html
<html data-theme="dark">
  …
</html>
```

To avoid a flash, set it before paint with a tiny inline script that reads the
user's stored choice or their system preference — exactly what this site does.

## Render a component

Import any component and render it. Props like `variant`, `tone` and `size` map to
the public `data-*` anatomy that you can target with your own CSS.

```tsx
import { Button } from '@manti-ui/react';

export function Example() {
  return (
    <Button variant="solid" tone="primary">
      Knead dough
    </Button>
  );
}
```
