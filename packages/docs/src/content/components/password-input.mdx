# PasswordInput

A password field with a show/hide toggle and a live Caps Lock warning. It is a
static component that renders the shared `field` shell — so label association,
`aria-describedby` / `aria-invalid`, sizing, and the focus ring behave exactly
like [TextField](/components/text-field) — and layers the password affordances
on top. It accepts any native input attribute.

<Demo name="password-input/basic" />

## States

`error` flips the field to the invalid state and replaces the hint,
`defaultVisible` starts with the text revealed, and native flags like `disabled`
work as expected. The Caps Lock warning appears automatically while typing with
Caps Lock on.

## Anatomy

PasswordInput reuses the `field` shell, so its core anatomy and component tokens
are the field's:

It adds two parts of its own under the `password-input` scope:

- `[data-scope="password-input"][data-part="visibility-trigger"]` — the show/hide button.
- `[data-scope="password-input"][data-part="caps-lock"]` — the Caps Lock notice.
