# Button

The workhorse action. Four variants across every tone, three sizes, a smooth
press, and a loading state that keeps the button from collapsing.

<Demo name="button/variants" center />

## Variants

`variant` sets the visual emphasis: `solid`, `soft`, `outline`, or `ghost`.

<Demo name="button/variants" />

## Tones

`tone` carries semantic meaning. Built-in tones are `primary`, `neutral`,
`success`, `warning`, `danger`, and `info` — and any custom tone works too.

<Demo name="button/tones" />

## Sizes

<Demo name="button/sizes" />

## Loading

`loading` shows a spinner, blocks interaction, and preserves the button's width
so the layout never jumps.
